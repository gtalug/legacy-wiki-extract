  • Leah : Site updated to MediaWiki to 1.5.3 version after DB hack of doom and various problems with an upgrade occurred. Things should be about back to normal. Let us know if anything is not working as it used to be.
October 23rd, 2005
  • Sy: A specifically-crafted mechanism has been crafted to thwart certain kinds of spamming. It can be adjusted easily as the need arises.
October 21st, 2005
  • User:MusaGum, our favourite spammer, has returned after creating a username. (see Special:Contributions/MusaGum)
Wednesday October 5th, 2005
  • Leah fixed the mediawiki skin for GTALUG again. You may find that when you log in to edit the site that you are getting the old page skin. This is a bug that happens when you change skins for a new installation some times. If you would like to see the pages with the gtalug skin, go to Special:Preferences when logged in. Select Skin, then change the skin to the GTALUG skin and select Save preferences.
  • Leah I have changed the skin to be a gtalug skin, to help avoid problems with updates in the future so that it is separate from the default skin. It is available at http://gtalug.org/skins/gtalug.tgz
  • Leah Additional work cleaning up the css for this page to look a bit nicer. Let me know if you have a problem viewing the site with the changes. Rounded borders were added, but only are supported for mozilla, so unknown how the site looks in IE.
Tuesday October 4th, 2005
  • Sy enabled the spam blacklisting extension, and things are set up to automatically update via mediawiki's spam blacklist. Unfortunately, bugs or limitations with the extension disallow the use of the excellent http://chongqed.org/ blacklist for now.
  • The spammer's existance has been erased from history. Mandatory re-education volunteering will be made available for anyone who knows about the event.
Monday October 3rd, 2005
  • Sy updated MediaWiki, unfortunately displacing all of the wiki customization (doh!).
  • Experimentation with spam prevention has been a moderate failure. Apparently black magic must be involved to make it work.

Fri May 27

  • Toronto > TTC > TTC subway map was donated by Sy. Similar resources will eventually find themselves here, as Toronto is a pretty significant hub, and it's useful to have info like this for visitors.
Sun May 15
  • Administration was created to house group policies and the like. The more structure-related stuff will go here.
  • Main Page's "Active wiki topics" has been fleshed out.
  • A proposal for open accounting has been created.
  • Some adjustments have been made to separate Resources (real world resources) from Computing (computing resources) although there is overlap with Contact. Well.. it'll get sorted out in time. For now it's important to make things findable.

  • Now that each presenter has a page, we can now generate a decent list of presenters!
  • Historical:Meetings has most of its meeting descriptions completed.
  • Some group history is still outstanding.. our mailing list archive resources are incomplete. Please see Talk:Historical:Meetings and GTALUG Archaeology and help if you can.
  • Issues was created to house Linux, free software and computing issues the group is interested in.
Fri May 13
  • About > Random was created to house random group trivia.
  • Additional work on Historical:Meetings continues.
Wed May 11
  • Drew Has back-filled most of the early years in Historical:Meetings
  • Seneca continues her efforts with Historical:Meetings, adding the older usergroup meeting announcements.
Tue May 10

Sun May 08

Tue May 03
  • Main Page has been unlocked so that it can be updated by anyone who has a change in announcement or another tidbit to add into this recent changes list. Graffiti concerns should be minimal at this point, as there's enough user activity here to do fast rollbacks as necessary. Additional troll/spammer protection is going to be added for apache itself, and other solutions are being researched preemptively. -- Sy
  • Psema4 and sub-topics has blossomed with activity over the last few days. Dl33t has done quite a lot of work with it.
  • Linux distributions has been mercylessly refactored into a nice clean table and sub-pages. Nice work Leah.
  • Wiki policies is being created, to reflect some decent practices. Right now it's a bunch of Sy's opinions, but in the future it'll be tidied up and items can be voted on as necessary.. the future and value of this page depends on the spirit and culture which forms here, and how things evolve in the future.
